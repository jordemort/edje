\subsection{Detailed Description}
Functions to deal with edje internal object. 

Don't use in standard situations. The use of any of the edje\_\-edit\_\-$\ast$ functions can break your theme ability, remember that the program must be separated from the interface!

The API can be used to query or set every part of an edje object in real time. You can manage every aspect of parts, part states, programs, script and whatever is contained in the edje file. For a reference of what all parameter means look at the complete \hyperlink{edcref}{Edje Data Collection reference}.

All the functions that deals with part states include the state value inside the returned strings (ex: \char`\"{}defaut 0.00\char`\"{}). For this reason there aren't functions to set/get a state value, you have to pass the value inside the name string (always in the form \char`\"{}state x.xx\char`\"{}).

Don't forget to free all the strings and the lists returned by any edje\_\-edit\_\-$\ast$() functions using \hyperlink{Edje__Edit_8h_924beb7b18a2f754b4b78cee3b3cc662}{edje\_\-edit\_\-string\_\-free()} and \hyperlink{Edje__Edit_8h_68ea52098aa643bf273436cf04e92ccf}{edje\_\-edit\_\-string\_\-list\_\-free()} when you don't need anymore.

Example: print all the part in a loaded edje\_\-object 

\begin{Code}\begin{verbatim}  Evas_List *parts, *l;

  parts = edje_edit_parts_list_get(edje_object);
  while(l = parts; l; l = l->next)
  {
     printf("Part: %s\n", (char*)l->data);
  }
  edje_edit_string_list_free(parts);
\end{verbatim}
\end{Code}



Example: Change the color of a rect inside an edje file 

\begin{Code}\begin{verbatim} Evas_Object *edje;

 edje = edje_object_add(evas);
 edje_object_file_set(edje,"edj/file/name", "group to load");
 edje_edit_state_color_set(edje, "MyRectName", "default 0.00", 255, 255, 0, 255);
 edje_edit_save(edje);
\end{verbatim}
\end{Code}

This function add the given image inside the edje. Don't add a new image part but only put the image inside the edje file. It actually write directly to the file so you don't have to save (and you can't undo!). After you have to create a new image\_\-part that use this image. Note that all the parts in the edje share the same image collection, thus you can/must use the same image for different part.

The format of the image files that can be loaded depend on the evas engine on your system\begin{Desc}
